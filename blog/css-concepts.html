CSS Sticky Notes: Class and ID Selectors

CSS is a markup language considered highly valuable in the realm of web development because of the organizational heirarchy it provides in separating all the design and formatting "skin" elements from the HTML "skeleton" of a website. In this sense, a designer can easily and efficiently navigate to different layers of a working project, to tweak structure or style accordingly and seeing how they would coexist (or clash) in a live construct of the site. CSS has it's own set of easy to follow syntax rules that allow these systematic changes, to either individual or grouped elements: two of the most invaluable CSS selector tags for doing this are the "class" tag and the "id" tag. I like to think of "class" and "id" as sticky notes reminding your computer to interpret design changes and here's why:

The "class" selector acts as a sticky note in CSS that consolidates a change/design you want across multiple elements of a certain type, and labels that change for the group to be interpreted in HTML. Often you do not want to apply the same styling rules to several selectors across your document over and over again in HTML. Rather, you can apply a "class" label to all those elements in HTML and then define the styling for that class in your CSS file that is linked to the document. Say we have some 'div' sections in HTML with several paragraph sections below them. Instead of styling each paragraph, we can assign different parent-div classes with a label (in this case, "fancy"). We can then go to our CSS file and use the syntax "." + "fancy" followed by "{ font-family: cursive; }" to apply a cursive font to all paragraphs with this class label. How easy is that?!

Sometimes we want a sticky note that tells us we have very specific styling needs for an individual element in our HTML/CSS file. In this case, we would apply the "ID" label. IDs are assigned to elements in HTML with the word "id" and an equal sign, not unlike the process for assigning a "class". The main difference in syntax comes when you are defining your ID in CSS; while you used a "." to define a class, you would use a "#" to associate your id and its attributes. As you can see in the example to the right, though we have multiple HTML paragraphs in this specific 'div' group, we associated an ID only to the first paragraph. That paragraph alone will receive the formatting we have designated in our linked CSS file.

As it should be apparent to you now, Class and ID selectors are an important aspect of the CSS workflow, intended to make your partitioned format choices organized and that much easier to find. In the event you want to overhaul your design choices, all you have to do is refer back to your CSS file and find the correct ID or Class that you used to apply the style.
