An Intro To Instance Methods and Variables

Methods that are available on classes are called class methods while those available on instances are called instance methods. Instance methods are methods that are defined inside the class body.

We can create a new instance from the Calculator class and call the plus instance method on it like the example below. As you can see this new calculator object has inherited several instance methods that now know how to function as a basic calculator.

Classes are objects in Ruby. This means that like other objects they can do do/receive things according to the methods they are indebted with (as in the behavior explained above), but like other objects they also know stuff. This can be done with he class method ‘initialize’, which bestows the class-object with data. When you call ‘new’ on the class Person, and pass the string "Ada" then the method ‘new’ will create a new instance of the class, and call initialize on it, passing the same argument list, which in our case is the single string “Ada". This “person” knows its own name, “Ada”!

When we create different variables inside of a class not every method can see the variables that we use. If we want to make a variable visible to different methods inside of a class, we call it an instance variable (indicated by the @ sign). That means that the variable should be visible to the whole instance of the class. If we want a variable that is only visible inside of the method or block that we are working with, that is called a local variable. This whole concept is called the “scope” of a variable. A classes scope is populated with instance variables, in the moment we assign something to them. And they are visible everywhere in this class, that is, in every method that the object has.
